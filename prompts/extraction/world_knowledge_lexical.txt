Purpose:
This file is only a schema-driven lexical aid. It may help choose among symbols that already appear in KB_SCHEMA. It is not additional vocabulary.

Rules:
- Never invent a predicate, function, type, or constant from this file.
- Never output a symbol unless it exists in KB_SCHEMA.
- Never output a constant unless it is supported by the case text or already present in normalized case entities.
- If this file conflicts with KB_SCHEMA, the case text, or the question, ignore this file.

Generic disambiguation patterns:
- Legal text often distinguishes actors, objects, acts, events, decisions, documents, periods, procedures, and relationships. Match the role order required by the schema.
- A legal role or status in ordinary language may map to a predicate over a broader type, not to a separate type.
- Numeric wording usually maps to a function/value assertion if the schema has a matching function.
- Legal conclusions should usually map to derived predicates during query extraction, not to observable facts during case extraction.
